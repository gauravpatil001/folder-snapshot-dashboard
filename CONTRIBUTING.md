# Contributing

Thanks for your interest in contributing.

## How to Contribute

1. Fork the repository.
2. Create a feature branch from `main`.
3. Make focused, testable changes.
4. Run a local build before opening a pull request.
5. Submit a pull request with a clear summary.

## Local Development

```powershell
cd FolderSnapshotDashboard
dotnet build
dotnet run
```

## Pull Request Guidelines

- Keep changes small and focused.
- Include a clear description of what changed and why.
- If UI behavior changes, include screenshots.
- Update `README.md` when behavior or setup changes.

## Coding Standards

- Use clear names and keep methods concise.
- Prefer simple, maintainable solutions.
- Do not include unrelated formatting-only changes.

## Reporting Issues

When opening an issue, include:
- Steps to reproduce
- Expected behavior
- Actual behavior
- Environment details (Windows version, .NET SDK version)
